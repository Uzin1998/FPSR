# 设计操作系统 v0.3 卡片模板规格

日期：2026-07-22  
状态：历史补档 / 待用户书面复核
规则来源：RULE-SRC-0001 brainstorming-spec-review-gate

## 1. 目标

v0.3 将 v0.1-v0.2 的原则推进到固定承载单元。

核心改动：

```text
规范不再只是一组原则，而是由固定字段的卡片承载。
```

v0.3 定义五类核心卡：

```text
系统契约卡
对象规格卡
体验锚点卡
测试定义卡
关系边卡
```

这些卡片在 v0.4.1 中进一步被投影为 Markdown + YAML Front Matter。

## 2. 共享元字段

所有卡片共享：

```text
ID
名称
责任面
状态
版本
来源
依赖
缺口
冲突
变更记录
```

规则：

```text
未知字段必须显式写 UNKNOWN / TBD / N/A
ID 永不复用
名称可改，ID 不改
来源必须可追溯
缺口和冲突不得混写
```

## 3. 系统契约卡

用途：

```text
承载规则、生成器、约束外壳、流程产生器。
```

字段：

```text
ID：
名称：
责任面：
状态：
版本：
来源：
依赖：
缺口：
冲突：
变更记录：

目的：
输入：
输出：
状态变更：
硬约束：
软目标：
随机源：
数据引用：
反模式 / 禁止特征：
测试引用：
开放问题：
```

边界：

```text
系统契约卡可以描述规则和逻辑。
系统契约卡不能伪装成 UE 实现。
系统契约卡不能跳过测试指标。
```

## 4. 对象规格卡

用途：

```text
承载物资、道具、敌人、房间、交互物等数据事实。
```

对象卡只能引用系统，不能暗藏规则。

字段：

```text
ID：
名称：
责任面：
状态：
版本：
来源：
依赖：
缺口：
冲突：
变更记录：

对象类型：
标签：
参数：
所属系统：
可参与关系：
表现需求：
特殊钩子：
禁止内置逻辑：
测试引用：
```

边界：

```text
对象卡声明数据、标签、参数、表现需求和特殊钩子。
具体规则必须回到系统契约卡。
```

## 5. 体验锚点卡

用途：

```text
把体验预期、高光、禁区转成可约束内容。
```

字段：

```text
ID：
名称：
责任面：
状态：
版本：
来源：
依赖：
缺口：
冲突：
变更记录：

正向目标：
禁止特征：
理想高光：
适用场景：
代理指标：
人工验收：
关联系统：
优先级：
```

规则：

```text
正向目标描述要产生什么
禁止特征描述绝不能生成什么
代理指标描述可统计观察信号
人工验收描述必须试玩或观察的信号
```

## 6. 测试定义卡

用途：

```text
定义测试模块该怎么读规格、跑仿真、判失败。
```

字段：

```text
ID：
名称：
责任面：
状态：
版本：
来源：
依赖：
缺口：
冲突：
变更记录：

测试目标：
读取规格：
输入状态：
玩家模型：
样本量：
指标：
通过阈值：
失败报告格式：
禁止修复方式：
回归范围：
```

边界：

```text
测试定义卡不是测试脚本。
测试定义卡不直接修改规范。
测试定义卡必须声明失败报告格式和禁止修复方式。
```

## 8. 最小规范闭环

v0.3 的最小闭环：

```text
体验锚点卡
-> 系统契约卡
-> 对象规格卡
-> 关系边卡
-> 结构化规格
-> 测试定义卡
-> 测试报告 / 缺口 / 冲突 / 返工
```

## 9. 禁止事项

```text
不把对象卡写成隐藏脚本
不把测试卡写成实现代码
不把体验锚点伪装成数学定理
不让关系边只停留在图示
不让 UNKNOWN / TBD 从字段中消失
不跳过来源和变更记录
```

## 10. v0.3 限制

v0.3 只定义卡片字段和职责。

它不定义：

```text
YAML Front Matter
raw source custody
测试报告字段
外部资料引入 skill
UE 实现结构
```

这些由 v0.4.1、v0.5 和后续工作补充。
