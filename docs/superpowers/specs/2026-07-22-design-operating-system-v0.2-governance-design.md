# 设计操作系统 v0.2 治理协议

日期：2026-07-22  
状态：历史补档 / 待用户书面复核
规则来源：RULE-SRC-0001 brainstorming-spec-review-gate

## 1. 目标

v0.2 修正 v0.1 中容易被误解的“层级模型”，并补充 ID、状态、校准、测试回流等治理协议。

核心修正：

```text
规范文档 = 语义真相源
结构化规格 = 经过校准的机器可读投影
测试模块 = 证据 / 报告 / 建议补丁生成器
代码 = 已定型规范的实现，不反向决定设计
```

## 2. 五个责任面

v0.2 将“五层模型”改名为“五个责任面”，避免机械上下分层。

```text
R0 体验锚点：目标、禁区、高光、验收信号
R1 元系统：流程产生器、规则模具、约束外壳
R2 关系模型：类型化关系图 / 超图、可达性视图
R3 概率数值：随机变量、分布、仿真指标
R4 实现适配：UE、输入、表现、资产、工程测试
```

R2 与 R3 可以交叉。概率可以是：

```text
节点属性
边属性
系统参数
运行时状态
条件修正
采样过程
```

## 3. ID 协议

所有可引用对象必须有稳定 ID。

初始 ID 类型：

```text
ANCHOR-0001    体验锚点
SYS-0001       系统契约卡
OBJ-0001       对象规格卡
EDGE-0001      关系边
BATCH-0001     资料批次
SRC-0001       源文件
GAP-0001       缺口
CONFLICT-0001  冲突
TEST-0001      测试定义
REPORT-0001    测试报告
PATCH-0001     建议补丁
```

规则：

```text
ID 永不复用
名称可改，ID 不改
任何推理、测试、代码生成都必须引用 ID
未知内容不能省略，必须显式标记 UNKNOWN / TBD / N/A
```

## 4. 状态机

v0.2 定义了较细的生命周期状态。

```text
候选
待查阅
发散中
契约建模
待结构化
待校准
待验证
已定型
返工中
冻结
废弃
```

后续 v0.4.1 为兼容 archive，将这些细状态拆分为：

```text
顶层状态：UNREAD / TRIAGED / OPEN / HYPOTHESIS / TESTING / LOCKED / REWORK / REJECTED
机器字段：projection_status / verification_status / calibration.status
```

## 5. 状态迁移

核心迁移：

```text
候选 -> 发散中：确认有设计价值
发散中 -> 契约建模：边界足够清楚
契约建模 -> 待结构化：规范字段基本完整
待结构化 -> 待校准：机器投影已生成
待校准 -> 待验证：语义映射通过
待验证 -> 已定型：证据 / 仿真 / 人工验收通过
已定型 -> 返工中：出现冲突、新证据或测试失败
返工中 -> 待验证：修改完成，进入回归
任意 -> 冻结 / 废弃：明确暂停或淘汰
```

## 6. 校准协议

结构化规格不能自动生效。每次从 Markdown 投影到 JSON、YAML 或表格，必须记录：

```text
来源文档 ID
投影版本
字段映射依据
不确定字段
人工 / 规则校准状态
允许进入测试的版本号
```

校准失败时：

```text
不允许进入测试
生成缺口或冲突
不修改原始资料
不自动覆盖 Markdown 正文
```

## 7. 测试闭环

测试模块不能改规范，但可以生成：

```text
失败证据
指标报告
疑似责任面
禁止修复方式
建议补丁
候选参数
返工建议
```

任何修改必须回到规范文档，再重新生成结构化投影、重新校准、重新测试。

## 8. 最小可运行闭环

v0.2 定义第一版“跑起来”为抽象闭环：

```text
规范文档
-> 结构化规格
-> 语义校准
-> 初始化抽象状态
-> 决策步推进
-> 多玩家模型仿真
-> 指标报告
-> 缺口 / 冲突 / 返工队列
```

这不是 UE 原型，也不是完整游戏手感。它是游戏系统的可验证核心。

## 9. 体验进入测试

R0 体验锚点必须拆成三段：

```text
正向目标
禁止特征
验收信号
```

验收信号分两类：

```text
代理指标：可由仿真统计验证
人工验收：必须试玩、观察或主观判断
```

## 10. 返工协议

已定型条目允许返工，但必须走流程：

```text
触发证据
影响范围
替代方案
回归测试
裁决记录
```

## 11. v0.2 限制

v0.2 仍未定义：

```text
卡片模板
raw source custody
YAML Front Matter 格式
测试报告字段
外部资料引入器 skill
```

这些由 v0.3-v0.5 补充。
