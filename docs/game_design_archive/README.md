# Game Design Archive

本目录用于收纳、分析并演化当前房间式肉鸽项目的底层设计。

当前归档对象：2026-07-21 本次对话中的核心想法、分析结论、风险点、修改办法和可拓展方向。

## 目录结构

```text
00_raw_sources/
  原始源文件副本库。保存喂入资料的不可变副本、批次 manifest 和 SHA-256。

00_inbox/
  新增资料工作入口。所有尚未查阅、尚未拆分的材料索引、转写版或拆分任务先放这里。

01_conversation_archive/
  对话记录、原始想法摘录、上下文来源。

02_analysis/
  已分析内容，包括核心判断、风险、不合理点、修改办法、拓展方向。
  评审类文档使用独立活跃性标记，见 02_analysis/README.md。

03_system_registry/
  系统清单、系统状态、依赖关系、待设计接口。

04_axioms_and_constraints/
  已定型原则、硬约束、禁止项、设计钉子。

05_models/
  逻辑模型、数据模型、有向图模型、概率模型、测试模型。

06_rule_sources/
  规则来源索引。记录外部 skill 流程门、状态语句和约束语句的来源与适用范围。
```

## 状态标记

```text
[UNREAD]     新加入，尚未查阅。
[TRIAGED]    已查阅，已拆分。
[OPEN]       未定型，可继续发散。
[HYPOTHESIS] 有设计假设，需要论证或验证。
[TESTING]    需要仿真、样本统计或玩法验证。
[LOCKED]     已定型，默认不随意修改。
[REWORK]     已发现问题，需要返工。
[REJECTED]   明确废弃。
```

## 文档活跃性标记

评审、审查、复盘和裁决迁移报告具有强时效性，使用独立活跃性轴，不并入上面的设计成熟度状态机。

规则来源：`RULE-SRC-0003 review-document-lifecycle`

```text
[ACTIVE]          当前仍驱动待办、修复、裁决或决策。
[CLOSED]          本轮审查已处理完或完成迁移，保留为历史证据。
[SUPERSEDED]      已被后续文档取代，引用时必须跳转到后继入口。
[REFERENCE_ONLY]  仅作背景材料，不承担行动义务。
```

边界：

```text
[LOCKED] 不表示评审文档已不活跃。
[REJECTED] 不表示评审文档已过期。
活跃性不用于系统卡、对象卡、测试卡的设计成熟度。
```

## 后续资料进入规则

1. 新资料先进入 `00_raw_sources/`，按 `BATCH-*` 建立源文件副本和 `manifest.md`。
2. 每个源文件必须分配 `SRC-*`，并记录 SHA-256。
3. `00_inbox/` 只保存待查阅工作入口，例如源文件索引、转写版、摘要或拆分任务。
4. inbox 文件头部必须标记状态，默认 `[UNREAD]`。
5. 查阅后拆成三类：已定型约束、未定型问题、可测试假设。
6. 查阅拆分完成后，把 inbox 工作入口改为 `[TRIAGED]`，并记录处理去向。
7. 不直接把未读资料或 raw source 写进定型设计，避免污染已锁定原则。
