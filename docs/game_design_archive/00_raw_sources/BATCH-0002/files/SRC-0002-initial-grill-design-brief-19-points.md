# 初始 Grill 设计纲领 19 条

## 原始内容

1. 收纳已有资料和设计（暂时没提供，先留好位置，约定持续区分新旧内容）
2. 系统化，推理，理顺
3. 保留拓展和发散能力，定型的标记好，未定型的持续发散拓展
4. 落逻辑系统和纯数据系统，在这个基础上才有 UE 游戏内具体实现
5. 基本物资和经济单位已定好，部分设计完全，部分只占位
6. 流转单元和交互接口没定死且仍想拓展
7. 玩法和玩家技术也是一种抽象资源
8. 以撒为标杆，最起码做到以撒级别，最好超越
9. 定了具体对象、单位，体验预期，表现预期，一些特殊情景和理想高光。底层系统填充需要 AI 发力，甚至可能需要数学建模。主要是结果和测试驱动，经验和数学、算法、工程设计减少弯路
10. 游戏不是固定流程，底层实现需要的是元能力，不是具体流程本身而是流程产生器。重点不在内容本身而在限制和系统外壳
11. 比喻：我知道我要什么塑像，你不需要把黏土塑形成某个固定的形状。你需要做的是把模具做好，甚至提供一组模具，然后在范围内打上钉子，这些钉子是进一步限制这个塑像不该有哪些特征
12. 物资道具经济对象交互物等能画出全局全可能性有向图。这个有向图是静态的，在此基础上考虑时序和空间还有房间推进，给图加上当前可达性这一属性，得到图的变化序列。加上推理时对玩家情况的分类讨论，是以有向图为结点的树状结构
13. 另一种做法：保持全局有向图，另外再维护一个局部有向图的概念，表示当前可转换可达的对象
14. 随机是根本立脚点
15. 有多种随机概念：随机种子，纯随机，单局固定概率（每开一局变一次），道具 a 默认爆率 x，道具 b 在 80% 对局爆率为 w 其余为 y。有概率的概率，微积分思想。还有稳定性等。不是简单的随机，游戏乐趣复杂性和长期可玩性必然依赖深入的概率系统设计和统计学应用
16. 很多随机本身也是随机的。简单设计：高压房间爆率提高 50%。复杂设计：高压房间 85% 提高爆率 50%，15% 提高爆率 25%。更进一步：高压房间的爆率提升度随数值上升而上涨
17. 非开放世界的高自由度，自由的是玩法和行动，不是空旷空间和无意义道具。主要想要新鲜度和探索感。仍然是房间式肉鸽游戏
18. 后续会用技术手段支持海量脆皮怪给玩家爽刷。需要在游戏设计层面完美接入：是否有/有多少奖励，什么时候出现，出多少，如何与通常肉鸽敌人做区分（包括视觉）
19. 设计过程可看作若干个控制系统。目前确定了一种：推导链条，第一个系统需要第二第三个系统，这两个系统可拆新话题设计实现，最后得到系统间连线的庞大图谱。开发流应该知道已有系统、系统状态（提出需要/开发中/完毕/返工等），并支持记录并处理冒出的新系统
