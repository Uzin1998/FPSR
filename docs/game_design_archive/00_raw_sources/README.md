# Raw Sources Protocol

状态：`[LOCKED]`

本目录保存喂入资料的原始源文件副本。它是证据保全层，不是工作队列、分析区或规范区。

## 职责

```text
00_raw_sources/
  保存原始源文件副本。
  只增不改。
  记录批次、来源、文件清单和 SHA-256。

00_inbox/
  保存等待查阅和拆分的工作入口。
  可以引用 raw source，也可以保存转写版、摘要或拆分任务。
```

## 批次结构

每次喂入资料建立一个批次目录：

```text
BATCH-0001/
  manifest.md
  files/
    原始文件副本
```

批次目录名使用稳定 `BATCH-*` ID。ID 永不复用。

## manifest 格式

```text
批次ID：BATCH-0001
加入时间：YYYY-MM-DD HH:mm
来源：用户 / AI / 外部资料 / 测试结果
资料类型：灵感 / 局部设计 / 方向性文档 / 混合
接收说明：

文件清单：
- source_id: SRC-0001
  原始文件名：
  保存路径：
  SHA-256：
  文件类型：
  是否已建立 inbox 条目：否
  关联 inbox：
  备注：
```

## 规则

1. 源文件副本永不编辑。
2. 如果需要清洗、转写、摘要或拆分，产物进入 `00_inbox/` 或后续分析区，不覆盖 raw source。
3. 每个源文件必须有 `SRC-*` ID 和 SHA-256。
4. 分析、卡片和测试报告引用资料时，优先引用 `BATCH-*` / `SRC-*`。
5. 图片、PDF、音频、压缩包、Markdown、文本都先按源文件副本保存。
6. 如果源文件后来被证明无效，不删除原副本，只在 `manifest.md` 备注无效原因。
7. raw source 不能直接进入定型规范，必须先经过 inbox 查阅和拆分。

## 禁止规则

1. 不在本目录里写 AI 推导结论。
2. 不修改源文件内容以适配后续分析。
3. 不用同一个 `SRC-*` ID 指向两个不同文件。
4. 不把 inbox 工作副本反向当作原始来源。
5. 不让无 hash 的资料进入卡片 `sources`。
